# 要件定義書：記念日・誕生日リマインダー

**作成日**: 2026-07-04

## 一言まとめ

> **誰の**: 家族や友人の誕生日・記念日をよく忘れてしまう人
> **何を**: バラバラに覚えている大事な日を一箇所にまとめて、うっかり忘れを防ぐ
> **どう**: 名前と日付を登録するだけで、近い順に並び替え・直近7日は強調表示してくれるシンプルなWebアプリ

## 背景・課題

- 誕生日や記念日は手帳・LINE・記憶などに分散しがちで、気づいたら過ぎていることがある。
- カレンダーアプリに毎年登録し直すのも面倒。名前と日付を一度登録すれば、以降は「あと何日か」が自動で分かる状態にしたい。

## 機能要件

| ID | 機能名 | 説明 |
|----|--------|------|
| F-01 | 登録 | 名前・日付・種別（誕生日／記念日／その他）を入力して登録する |
| F-02 | 一覧表示 | 今日からの残り日数が近い順にソートして一覧表示する |
| F-03 | 強調表示 | 残り7日以内の項目を目立つ色・枠で強調表示する |
| F-04 | 種別ごとの色分け | 誕生日／記念日／その他で異なる色のバッジ・背景を表示する |
| F-05 | 編集 | 登録済みの項目を後から編集できる |
| F-06 | 削除 | 登録済みの項目を削除できる |

## スコープ外（今回はやらない）

- ブラウザ通知・LINE連携などの通知機能
- 複数デバイス間でのデータ同期・アカウント認証
- 繰り返し以外の特殊な日付パターン（例：「毎月◯日」のような月次リマインダー）

## データ仕様

**データモデル**
```
{
  id: string,
  name: string,       // 名前
  date: string,        // 記念日の月日（年は問わない）
  type: '誕生日' | '記念日' | 'その他'
}
```

**残り日数の計算ロジック**
- 登録された日付の「月日」を今年の日付とみなして今日と比較する
- すでに今年の日付を過ぎている場合は、来年の同じ月日として計算し直す（年をまたぐケースに対応）
- 計算結果（残り日数）の昇順で一覧をソートする

**保存方法**
- サーバー・DBは使わず、ブラウザの `localStorage` にすべて保存する（個人利用・単一デバイス前提）

## UI/デザイン方針

- 画面遷移なしのシングルページ構成（課題1と同様）
- 上部に登録フォーム（名前・日付・種別）、下部に一覧表示
- 一覧は残り日数の近い順。7日以内の項目は背景色や枠線で強調
- 種別ごとに異なるバッジ色（誕生日＝ピンク系、記念日＝青系、その他＝グレー系など）
- 各項目に「編集」「削除」ボタンを設置
