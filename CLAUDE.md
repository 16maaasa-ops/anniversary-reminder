# CLAUDE.md

このファイルは、リポジトリで作業する Claude Code (claude.ai/code) へのガイダンスを提供します。

> Next.js のバージョン固有の注意点は `AGENTS.md` を参照。

## プロジェクト概要

家族や友人の誕生日・記念日をよく忘れてしまう人向けの、個人利用の記念日リマインダー Web アプリ。名前と日付を登録するだけで、今日から近い順に並び替え、直近7日以内の項目を強調表示する。詳細な要件は `@requirements.md` を参照すること。機能追加や仕様変更の際は必ず requirements.md との整合性を確認する。

## 技術スタック

- **フレームワーク**: Next.js（App Router）
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **データ保存**: ブラウザの `localStorage`（DB・認証なし。個人・単一デバイス利用が前提）
- **デプロイ先**: Vercel（無料枠）

## 開発コマンド

```bash
npm run dev      # ローカル開発サーバー起動 (http://localhost:3000)
npm run build    # 本番ビルド
npm run lint     # ESLint
```

## アーキテクチャ

画面遷移なしのシングルページ構成。主要ファイルは以下。

- `app/page.tsx` — 登録フォーム・一覧表示・編集・削除をすべて同一ページで管理
- `app/lib/anniversary.ts`（想定）— 残り日数計算などのロジックを分離したユーティリティ
- API ルートやサーバー処理は存在しない（すべてクライアントサイドで完結）

### データモデル

```ts
type Anniversary = {
  id: string
  name: string
  date: string // 月日（例: "07-04"）。年は問わない
  type: '誕生日' | '記念日' | 'その他'
}
```

### 残り日数計算ロジック（重要な仕様）

- 登録された日付の「月日」を今年の日付とみなして今日と比較する
- すでに今年のその日付を過ぎている場合は、来年の同じ月日として計算し直す（年をまたぐケースに対応。例：12月に登録した「来年1月の記念日」も正しく近い将来として扱う）
- 一覧はこの残り日数の昇順でソートする
- 残り日数が 7 日以内の項目は強調表示する（背景色・枠線を変える）

### 種別ごとの色分け

- 誕生日＝ピンク系、記念日＝青系、その他＝グレー系のバッジ・背景色で表示する

## データ永続化

- `localStorage` に保存・読み込みする。`useEffect` でマウント時にロードし、一覧が変更されるたびに保存する
- サーバー・DB・認証は一切使わない（要件のスコープ外）

## スコープ外（実装しないこと）

- ブラウザ通知・LINE連携などの通知機能
- 複数デバイス間のデータ同期・アカウント認証
- 「毎月◯日」のような繰り返しパターン以外の特殊な日付指定

## 作業ルール（重要）

このプロジェクトは前回（商品名生成／メルカリ出品アシスタント）で「Claude Codeへの指示の出し方・Plan Modeの使い方」に一番苦労した反省を踏まえて進めている。以下を徹底すること。

- 機能追加やロジック変更など非自明な変更は、必ず Plan Mode で設計を提示し、承認を得てから実装する
- 一度に複数の機能を混ぜて指示・実装しない。1機能ずつ「指示→実装→動作確認」のサイクルを回す
- 仕様に迷ったら requirements.md を参照し、そこにない拡張（通知機能など）は勝手に追加しない
